Initial Launch Phase Payment Method

Note: For the initial launch phase, the Niqati platform will exclusively support bank transfers to the local payment gateway account. Additional payment methods will be implemented but hidden from users until future phases.

This strategic approach allows for a controlled launch while ensuring the payment infrastructure is fully secure and tested before expanding to additional payment options. The system architecture will be designed to accommodate all planned payment methods, with only bank transfers being visible and active during the initial phase.

Bank Transfer Payment Flow

Step 1: Order Placement
  • User selects products and proceeds to checkout
  • System generates a unique order ID
  • Order is placed in "Pending Payment" status
Step 2: Payment Instructions
  • System displays bank account details for transfer
  • User is provided with:
    • Bank name and branch
    • Account holder name
    • Account number/IBAN
    • Reference number (Order ID) to include in transfer
    • Exact amount to transfer (including any fees)
  • Payment instructions are also sent via email/SMS
  • Order remains valid for 24-48 hours awaiting payment
Step 3: Payment Confirmation
  • After making the transfer, user uploads proof of payment:
  • Options for confirmation:
    • Upload screenshot/image of transfer receipt
    • Enter transfer reference number
    • Contact customer service via WhatsApp/Telegram with order ID and transfer details
  • System updates order status to "Payment Verification Pending"
Step 4: Manual Verification
  • Payment gateway administrator receives notification of payment claim
  • Administrator verifies the transfer against bank records
  • Once confirmed, marks payment as "Verified"
  • System updates order status to "Processing"
Step 5: Order Fulfillment
  • System proceeds with digital product delivery
  • User receives digital codes/vouchers via preferred method
  • User account is updated with purchase history
  • Order is marked "Complete"

Future Payment Method Expansion

While the system infrastructure will be built to support all planned payment methods from the beginning, these additional options will be implemented but hidden during the initial launch. This approach ensures a smoother transition when expanding payment options in the future.

Phase 1: Initial Launch (Bank Transfer Only)

  • Only bank transfer option visible to users
  • Manual verification process by payment gateway administrators
  • Focus on building trust and ensuring smooth operations
  • Collection of user feedback on payment process
  • Monitoring of payment verification turnaround time

Phase 2: Primary Digital Payment Methods (3-6 months post-launch)

  • Activate credit/debit card processing
  • Enable major digital wallets (Apple Pay, Google Pay)
  • Integrate with primary local payment services (e.g., STC Pay in Saudi Arabia)
  • Bank transfer option still available
  • Automated payment confirmation for digital methods

Phase 3: Full Payment Ecosystem (6-12 months post-launch)

  • Complete integration of all regional payment methods
  • User wallet system with balance management
  • USDT settlement for merchants
  • Enhanced fraud detection and prevention
  • Streamlined checkout experience with saved payment methods
Important Implementation Notes:

Despite showing only bank transfers to users during the initial phase, the system should be built with the full payment architecture in place, including:

  • Database schema supporting all planned payment methods
  • API endpoints for all payment integrations (even if inactive)
  • Admin controls to activate/deactivate payment methods per region
  • UI components for all payment methods (hidden until activated)

This approach minimizes technical debt and allows for seamless activation of additional payment methods when ready.
