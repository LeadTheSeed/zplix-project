4.2 Complete Payment Flow

Step 1: Checkout Initialization
  • User reviews cart items and proceeds to checkout
  • System detects user's country based on profile/IP address
  • Prices are displayed in user's local currency
  • Available payment methods for user's country are shown
Step 2: Payment Method Selection
  • User selects preferred payment method:
    • Credit/Debit Card
    • Local payment method (e.g., Mada, Fawry, KNET)
    • Digital wallet (Apple Pay, Google Pay)
    • Bank transfer (where available)
    • Platform wallet balance (if sufficient)
Step 3: Payment Gateway Integration
  • System routes to appropriate payment gateway based on:
    • User's country
    • Selected payment method
    • Transaction amount
  • Payment gateway initialization with order details
  • User redirected to secure payment page if necessary
Step 4: Payment Processing
  • Payment gateway processes the transaction
  • 3D Secure authentication where required
  • Gateway validates payment and sends response
  • System receives webhook notification of payment status
Step 5: Order Confirmation
  • System confirms payment and updates order status
  • Generates unique transaction ID and order reference
  • Sends confirmation via email/SMS/WhatsApp
  • Updates inventory and user purchase history

4.3 Currency Conversion

The system will handle automatic currency conversion for cross-border transactions:

  • Real-time exchange rates from reliable providers (Fixer.io, Open Exchange Rates)
  • Base currency: USD for internal processing
  • Display currency: User's local currency
  • Settlement currency: Merchant's preferred currency or USDT
Important Compliance Notes:
  • All payment processing must comply with local regulations
  • PCI DSS compliance required for card processing
  • Transaction monitoring for fraud prevention
  • KYC requirements for high-value transactions

    • رصيد محفظة المنصة (إذا كان كافياً)

5. Cryptocurrency Integration | تكامل العملات المشفرة

5.1 USDT Wallet System

Niqati will implement a robust USDT (Tether) wallet system for merchant settlements, enabling seamless transfer of funds across borders without traditional banking limitations. The platform will support TRC-20 and ERC-20 USDT tokens.

USDT Settlement Architecture

Wallet Creation
  • Merchants can create a USDT wallet through the merchant portal
  • Each merchant receives a unique wallet address
  • Multi-signature (2-of-3) wallet architecture for security
  • Private keys stored securely using Hardware Security Modules (HSMs)
Settlement Process
  • Customer purchases product using local payment method
  • Platform processes payment and confirms transaction
  • System calculates settlement amount after platform fee
  • Local currency converted to USDT at current market rate
  • USDT transferred to merchant's wallet address
  • Transaction recorded on blockchain with transaction hash
  • Notification sent to merchant with settlement details
Withdrawal Options
  • Direct withdrawal to external wallet
  • Conversion to local currency and bank deposit (where available)
  • Reinvestment in platform inventory

5.2 Security Measures

  • Multi-Signature Wallets: 2-of-3 multi-sig requiring multiple approvals for transactions
  • Cold Storage: Majority of funds stored in cold wallets
  • Hot Wallets: Limited funds in hot wallets for immediate settlements
  • Transaction Monitoring: Real-time monitoring for suspicious activities
  • Withdrawal Limits: Daily/weekly/monthly limits based on account tier
  • Enhanced KYC: Additional verification for high-value transactions
  • Two-Factor Authentication: Required for all wallet operations
Implementation Considerations:
  • Preferred Network: TRC-20 (Tron) for lower fees and faster transactions
  • Alternative: ERC-20 (Ethereum) for broader compatibility
  • Gas fee management for efficient transaction processing
  • Blockchain integration with Web3.js and specialized libraries
  • Real-time USDT price feeds from reliable sources
  • Compliance with crypto regulations in each operating country

5.1 نظام محفظة USDT

ستقوم منصة نقاطي بتنفيذ نظام محفظة USDT (تيثر) قوي لتسويات التجار، مما يتيح نقل الأموال بسلاسة عبر الحدود دون قيود المصارف التقليدية. ستدعم المنصة رموز USDT على شبكات TRC-20 وERC-20.

هندسة تسوية USDT

إنشاء المحفظة
  • يمكن للتجار إنشاء محفظة USDT من خلال بوابة التاجر
  • يتلقى كل تاجر عنوان محفظة فريد
  • هندسة محفظة متعددة التوقيعات (2 من 3) للأمان
  • تخزين المفاتيح الخاصة بشكل آمن باستخدام وحدات أمان الأجهزة (HSMs)
عملية التسوية
  • يشتري العميل المنتج باستخدام طريقة دفع محلية
  • تعالج المنصة الدفع وتؤكد المعاملة
  • يحسب النظام مبلغ التسوية بعد رسوم المنصة
  • يتم تحويل العملة المحلية إلى USDT بسعر السوق الحالي
  • يتم تحويل USDT إلى عنوان محفظة التاجر
  • تسجل المعاملة على البلوكشين مع رمز المعاملة
  • يتم إرسال إشعار إلى التاجر مع تفاصيل التسوية
